# Учебный проект: ASP.NET Core MVC — Управление каталогом товаров

## Общие сведения
Данный проект представляет собой упрощенную реализацию веб-приложения на базе архитектуры **MVC (Model-View-Controller)** и технологии **ASP.NET Core**. Приложение предназначено для демонстрации базовых принципов создания информационных систем, работы с базами данных через ORM-системы и реализации CRUD-операций (создание, чтение, обновление, удаление).

## Стек технологий
*   **Framework:** .NET 8.0 / ASP.NET Core MVC.
*   **ORM:** Entity Framework Core.
*   **База данных:** SQLite (локальный файл `products.db`).
*   **Frontend:** Razor Pages (CSHTML), CSS3.
*   **Валидация:** jQuery Validation и Unobtrusive Validation.

## Структура проекта
Проект организован в соответствии со стандартными соглашениями именования ASP.NET Core:

*   **/Models**: Содержит класс `Product.cs`, описывающий сущность товара и правила валидации данных с помощью атрибутов (Data Annotations).
*   **/Controllers**: Содержит `ProductsController.cs`, который обрабатывает HTTP-запросы, взаимодействует с базой данных и возвращает соответствующие представления.
*   **/Data**: Содержит `ApplicationDbContext.cs` — контекст базы данных, отвечающий за связь моделей с таблицами БД.
*   **/Views**: Набор Razor-представлений для отображения интерфейса:
    *   `Index.cshtml`: Список всех товаров.
    *   `Create.cshtml`: Форма добавления нового товара.
    *   `Edit.cshtml`: Форма редактирования существующей записи.
    *   `Delete.cshtml`: Подтверждение удаления товара.
*   **/wwwroot**: Статические файлы (CSS-стили).
*   `Program.cs`: Входная точка приложения, где настраиваются сервисы, внедрение зависимостей (DI), подключение БД и маршрутизация.

## Основной функционал

### 1. Работа с данными (Entity Framework Core)
Приложение использует подход **Code First**. В классе `ApplicationDbContext` реализован вызов `Database.EnsureCreated()`, что позволяет автоматически создавать файл базы данных при первом запуске приложения, если он отсутствует.

### 2. Модель данных и валидация
Сущность `Product` включает поля:
*   `Id`: Первичный ключ.
*   `Name`: Обязательное поле (не более 100 символов).
*   `Price`: Обязательное десятичное поле с проверкой диапазона (от 0.01).
*   `Description`: Необязательное текстовое описание (до 500 символов).

Валидация реализована как на стороне сервера (проверка `ModelState.IsValid`), так и на стороне клиента с использованием библиотек jQuery.

### 3. Маршрутизация
В приложении настроен маршрут по умолчанию: `controller=Products/action=Index`. Это означает, что при запуске приложения пользователь сразу попадает на страницу списка товаров.

## Инструкция по запуску

1.  **Подготовка окружения:** Убедитесь, что установлена среда выполнения .NET SDK (версии 6.0 или выше) и IDE (Visual Studio 2022, JetBrains Rider или VS Code).
2.  **Восстановление зависимостей:** Выполните команду `dotnet restore` или позвольте IDE сделать это автоматически.
3.  **Запуск:** Запустите проект (F5 или `dotnet run`). База данных `products.db` будет создана автоматически в корневой папке проекта.
4.  **Просмотр:** Откройте браузер по адресу `https://localhost:[порт]`.

## Ключевые особенности реализации для изучения
*   **Dependency Injection:** Внедрение контекста данных в контроллер через конструктор.
*   **Tag Helpers:** Использование специализированных тегов (`asp-for`, `asp-action`, `asp-route-id`) для генерации HTML-кода.
*   **Асинхронное программирование:** Использование ключевых слов `async/await` при обращении к базе данных для обеспечения высокой производительности.
*   **Partial Views и Layout:** Использование общего шаблона `_Layout.cshtml` для соблюдения принципа DRY (Don't Repeat Yourself).
